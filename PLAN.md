## 1. What this is

**Reflex** is a cognitive testing & training web app, built as an offline-first
PWA. Five procedurally-generated tests measure reaction time, visual tracking,
inhibition, working memory, and spatial reasoning. Everything is local — no
accounts, no backend, no analytics. The product ethos is **"a measuring
instrument you trust"**: honest milliseconds, honest DNFs, disclosed device
latency, data that never leaves the device (footer literally says so).

- **Live:** https://reflex-6yb.pages.dev (Cloudflare Pages, auto-deploys from `main`)
- **Repo:** https://github.com/MartinDurdovic/reflex
- **Owner:** Martin (user). English UI now, **Slovak translation planned** —
  that's why every user-facing string lives in `src/lib/strings.ts`.

### Priorities (in order)
1. **Correctness of game mechanics and timing accuracy** — above everything.
2. Design identity (see §6) — distinctive, NOT generic "AI slop".
3. Everything else.

## 3. Timing & correctness rules (NON-NEGOTIABLE)

These are the project's constitution. Violating them is a bug even if nothing
visibly breaks:

1. **`performance.now()` for anything score-related.** Never `Date.now()`
   (that's only for the `timestamp` history field).
2. **Never trust `setTimeout` for scoring.** All score-critical scheduling goes
   through `scheduleAt()` (`src/engine/timing.ts`) — a rAF loop checking
   `performance.now()` against an absolute target. Sequences are scheduled on
   an **absolute timeline** (t0 + k·interval), so no cumulative drift.
3. **Press time = `event.timeStamp` from `pointerdown`** (closer to hardware
   than handler execution), with a sanity fallback to `performance.now()` if
   the browser reports a non-monotonic value. Never `click` (adds ~100ms).
4. **Physics uses a fixed timestep** (`fixedStepLoop`, accumulator pattern,
   120 steps/s) so identical speed on 60/120 Hz screens. Phase timers advance
   on **simulated time**, never wall time → pausing can't eat or skip time.
5. **Multi-touch is ignored:** every pointer handler starts with
   `if (!e.isPrimary) return;`.
6. **Tab hidden mid-trial → pause or abort, never a garbage score.**
   `GameShell` handles it (`pauseMode: 'pause' | 'abort'`). Reaction-style
   games abort; MOT/rotation pause. A game **starting** on an already-hidden
   page aborts immediately (rAF is frozen there; no visibilitychange will fire).
7. **Failures are recorded as `valid: false`** (jump start, zero hits). They
   show in history/counts but are excluded from bests and averages.
8. **Every attempt records `deviceType`** (`touch`/`mouse`/`pen`, from actual
   pointer events) because touchscreen latency makes scores non-comparable
   across input types. Reaction results show the latency disclaimer footnote.
9. UI chrome is locked (`body.game-locked`) during live trials so a fast first
   tap can't hit a nav link.

**Game module contract:** each game exports `run(ctx: PlayContext)`;
`PlayContext = { meta, difficulty, config, stage }`. Games build DOM inside the
fullscreen `.game-stage`, drive themselves via `GameShell`, save attempts, and
end with `renderResults()`. Wire new games in `registry.ts` + the `loaders` map
in `play/[game].astro`.

## 5. The five tests (mechanics that matter)

### 01 · Lights Out (`reaction-f1`) — the flagship
F1 start clone. 5 columns × 4 red lamps light left→right at 1000ms intervals
(absolute timeline), then a uniform random 200–3000ms hold, then all out in the
same frame — `tOut` = the rAF fire timestamp. Reaction = press − tOut. Tap
before lights-out = **JUMP START** → `valid:false`. Session = 5 starts, one
stored attempt per start (`params.sessionId` groups them); results show session
best / average / consistency (std dev) / jump starts. **No 3-2-1 countdown**
(`countdown:false`) — the light sequence is the countdown. Score unit: ms,
lower better.

### 02 · Ball Tracking (`mot`)
Pylyshyn MOT. Phases: reveal (targets solid amber + glow, big on-screen
countdown) → tracking (all balls identical, constant speed, elastic equal-mass
collisions, heading perturbation every ~0.5s, speed renormalized each step) →
select (tap exactly K balls, fat-finger hit radius, confirm) → **answer reveal**
(3.5s: real targets green, wrong picks red, player's picks keep blue ring) →
results. **No difficulty tiers — player-configured** via intro config fields,
persisted in settings key `mot.config`:
- total balls 2–20 (default 8)
- balls to track 1–floor(total/2) (dynamicMax; default 3)
- *advanced:* speed 1–10 (default 4), size 1–10 (default 5),
  tracking time 5–600s (default 8), start countdown 0.5–5s step 0.5 (default 2)
Score = accuracy % (correct/K). Old attempts from the removed staircase system
(score = level) still sit in some users' history — known cosmetic wart.

### 03 · Go / No-Go (`go-nogo`)
30 stimuli (22 go green / 8 no-go red, Fisher-Yates), ISI 800–2500ms, 800ms
response window; tap-vs-timeout races settled by a per-stimulus `resolved`
flag. Records avg RT (score, ms), false alarms, misses, premature taps. Zero
hits → DNF.

### 04 · Digit Span (`digit-span`)
800ms on / 200ms off, no immediate repeats, on-screen numpad only. Start at 4;
+1 per success; one retry per length (fresh sequence); second miss ends round.
Modes: forward / reverse (difficulty selector). Score = longest correct span.

### 05 · Mental Rotation (`rotation`)
Shepard-Metzler with **static views** (user's explicit choice — no spinning, no
drag). Polycubes are self-avoiding **non-touching path walks** (clean "arms":
2 endpoints, no branches, no folds), rejected if flat or **achiral** — chirality
verified against all 24 grid rotations so a mirrored trial is always
distinguishable (`polycube.ts`, unit-tested). Right view = same shape (50%) or
mirrored (50%); both get random orientations chosen by sampling 120 quaternions
and scoring occlusion (`viewScore`) so **every cube is visible from one POV**.
Answers: **SAME / DIFFERENT**. 10 trials; difficulties: 7/10/13 cubes, time
limit none/15s/8s (simulated time). Score = accuracy %.

## 6. Design identity — "chronograph / instrument"

The user explicitly rejected generic futuristic-AI aesthetics. The identity is
a **warm analog instrument**: stopwatches, oscilloscopes, timing panels.

### House rules (enforce on every change)
- **One accent — signal amber `#FFB000` — used for meaning only** (action,
  record, active state), never decoration.
- **No gradients. No glassmorphism/blur. No glow** — except the F1 lamps
  (diegetic: they're lamps). No purple/cyan on navy, ever.
- Hairline borders over shadows. Two radii only (6px / 3px).
- **Every number in tabular IBM Plex Mono with its unit** ("213 ms", never 213).
- Micro-labels: uppercase, letterspaced, dim ("engraved plate" style) — via
  CSS `text-transform`, so translations stay lowercase in strings.ts.
- Voice: terse instrument copy ("SESSION BEST", "DNF").
- Density comes from **functional detail** (readout strips, section rules,
  unit badges, graticules, nameplate footer), never ornament.
- One signature motion: 100ms press transition. No fades/parallax.

### Identity devices
- **Brand mark: five start lamps, four dim + last amber.** Used in header
  wordmark, footer, favicon.svg, and generated PNG icons. The name Reflex +
  five lights = the brand.
- Footer nameplate on every page: lamps · `REFLEX · MK I` · "All data stays on
  this device" (pinned to viewport bottom; body is a flex column — **flex items
  with `margin:0 auto` need explicit `width:100%`**, this bit us once).
- Faint dot graticule on page background (4.5% alpha); game stages stay solid.
- Home = numbered **ledger** rows (not cards): index, name, unit badge, short
  desc, sparkline, PB/LAST. Mobile-first grid areas; **nothing ever truncates**
  — text wraps, data drops to its own line, sparkline hides <360px.
- Palette/typography/spacing all live in `src/styles/theme.css` vars.

## 7. PWA / service-worker lifecycle (fragile — understand before touching)

- `generate-sw.mjs` emits a full precache manifest; cache name embeds a content
  hash. Cache-first for precached, runtime-cache for the rest, clean-URL
  navigation fallback.
- **Update flow:** new SW installs and *waits* (no automatic skipWaiting).
  The page (Layout.astro) promotes it via `postMessage('skip-waiting')` and
  reloads on `controllerchange` — **but never while `body.game-locked`**
  (guarded at both the promote and the reload). First-install claim is guarded
  with `hadController` so it doesn't reload-loop.
- History lesson: skipWaiting+claim+delete-old-cache **mid-session 404'd
  lazy-loaded chunks** once. Don't reintroduce that.
- Lighthouse (mobile): perf 100 / best-practices 100 / a11y 91 / SEO 90.
  Lighthouse 13 has no PWA category; installability = manifest + icons + SW.

## 8. Data model

```ts
interface Attempt {
  testId: string;
  timestamp: number;        // Date.now(), history only
  score: number;            // per-game meaning (ms | % | digits)
  valid: boolean;           // false = DNF, excluded from best/avg
  difficulty: string;       // 'standard' | 'forward' | 'custom' | …
  params: Record<string, unknown>;  // full per-attempt detail (config, sub-metrics)
  deviceType: 'touch' | 'mouse' | 'pen' | 'unknown';
}
```
Storage keys: `attempts:<testId>` (array), `settings` (object; includes
`<gameId>.config`). Export = JSON blob (`app:'brain'` legacy tag — keep for
import compat); import merges + dedupes by timestamp.

## 10. Manual test checklist (after risky changes)

1. Install prompt appears (Android Chrome); app works fully in airplane mode
   after first load.
2. Lights Out: deliberate early tap → JUMP START → recorded as DNF; second
   finger during reaction does nothing; session stats sane.
3. MOT: config changes persist across reloads; targets clamp to half of total;
   answer reveal shows green/red truthfully; tab-switch mid-tracking pauses.
4. Go/No-Go: red tap = false alarm; >800ms green = miss.
5. Digit Span: retry-then-fail ends round; reverse mode expects backwards.
6. Rotation: shapes static & fully visible; SAME/DIFFERENT always answerable
   (chirality guarantees it); hard mode 8s timer runs on simulated time.
7. Settings: export → reset → import restores everything.
8. Deploy: new version auto-applies on next visit *unless* mid-game.
